# Tasks — Phase 5: Ecosystem & Governance (ongoing)

*Goal: make Ajar bigger than its authors. Reference: `ROADMAP.md` Phase 5. These are workstreams more than tickets; spawn concrete tasks per quarter.*

---

### T5.1 — Standing mandates & org↔org (the invoice scenario)
Recurrence-capped mandates between organizations; Gateway `contracted` tier flows; reconciliation tooling both sides. **DoD:** two real organizations run a recurring invoice/payment cycle with zero human touches per cycle, receipts reconciling automatically.

### T5.2 — A2A interop
Bridge: Ajar sites/actions discoverable and invocable from A2A agent ecosystems; agent cards ↔ manifest mapping notes. **DoD:** an A2A-native agent completes a task on an Ajar site through the bridge.

### T5.3 — Additional Gateway shapes
Shopify app; edge-worker deployment; static-site generator plugins (Hugo/Next/Astro emit views+manifest at build); hosting-panel one-click installs. **DoD:** each shape has docs + demo; parity with reverse-proxy conformance.

### T5.4 — Framework adapters
Agent-framework adapters over the Kernel (all inherit Monitor enforcement; adapters never bypass). **DoD:** quickstart per framework; enforcement-parity tests.

### T5.5 — Protocol interop bridges
WebMCP (manifest points at in-page tools), NLWeb (`/ask` as declared R0 action), ANP (agent-description mapping), ERC-8004 (optional identity anchoring). **DoD:** interop notes published per protocol; at least WebMCP + NLWeb bridges working.

### T5.6 — Spec v1.0 hardening
Fold field feedback; resolve OQ-1..5; freeze v1.0 with formal deprecation policy; conformance suite v1.0. **DoD:** two independent non-reference implementations pass conformance.

### T5.7 — Governance transfer
Submit spec to neutral venue (W3C CG and/or Linux Foundation per ADR-012); charter, IP policy, maintainer structure with external merge rights; trademark policy for "Ajar"/conformance badge. **DoD:** spec governance formally lodged outside the founding team; ≥2 external maintainers active.

### T5.8 — Adoption program
"Agent-ready" badge + public conformance directory; registrar/hoster partnerships; owner-facing education (what to expose, how to price); case studies from pilots. **DoD:** directory live; ≥3 external production Gateways; published case studies with numbers.

### T5.9 — Research track
OQ-4 privacy-preserving discovery (OPRF/PSI prototypes); probabilistic SIMULATE (OQ-3); reputation layers over receipts; TEE-attested Kernel builds. **DoD:** one published prototype/paper per year feeding the spec.

---

**Directional exit (project maturity):** ROADMAP Phase-5 criteria — external production deployments, external maintainers, neutral governance, and one real org↔org standing-mandate deployment.
